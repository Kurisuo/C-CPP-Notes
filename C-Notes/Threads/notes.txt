- pthread_init(&t1, NULL, &func, NULL)

- System waits for first Thread to finish execution and be collected by pthread_join(), only then does the next Thread recieve the lock to access the critical region.
- Is parallelism still existent if we are forced to wait for a thread to finish execution before letting another thread continue into the critical region?
    - Not recommended to use for-loop strategy implemented within forloopthreading.collected
