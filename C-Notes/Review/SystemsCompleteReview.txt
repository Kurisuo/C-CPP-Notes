- C variables (local, static, global)
    --> Local Variables: Of local scope - Live on the stack temporarily while being called, flushed afterwards when no longer in use
    --> Static: Lifetime of a static variable lasts until end of program. Value of a static does not end upon a function returning its value, set to NULL or 0 when not initialized
    --> Global: Program-wide scope, auto set to 0
    Ex1: Inside a function, consider char *p; and static char *q; before either is explicitly assigned. Which is correct?

##  Security


Threat Models - Always assume the attackers know what program you are running
    |--> Never rely on security through obscurity
    |Ex --> One secret key that holds access to all gates, if someone finds it security is completely compromised

Kerkchoff's doctrine: Security should depend on the KEY; count it as system design 
Attacker Motivation: Learn something privileged about the user/take over their machine

Attacker power:

Single-node: observe timing of actions, observe inputs/outputs, control inputs.
Multi-node (Man-in-the-Middle): observe, drop, and change messages between nodes.

Attacker lack of power: What we know they cannot directly do
    |--> Read program memory?
    |--> Decrypt properly encrypted data?

Corner-Case Bugs: When a system neglects logic on one or more corner cases, causing attackers to access them and exploit them
    |Ex--> Apple's goto fail which duped a goto line causing the program to skip sslRawVerify() authentication 
    |Solution --> Better testing & formal verification
    |Mitigation: Principle of least privilage, isolate components

Unsanitized-Input Bugs: System taking in user input without verifying it
    |Ex: SQL Injection

Side-Channel Bugs: Factors stemming from inputs/outputs that an attacker can use to find out needed information
    | Ex: Timing - Password comparison: strncmp() reports failure on the first char, exposing the password one char at a time
    | Ex2: Cache - attacker fills cache lets victim idle then times accesses to every cache line

Spectre: Speculative exclusion, mistrain the branch predictor

Meltdown: Exploits race between memory access and privilage checking

Dependency Vulnerabilities: Systems rely on internal software, break the dependency, break the system
Operational Errors: A system is only as secure as its operators
